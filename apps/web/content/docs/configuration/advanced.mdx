## Layer Rules

Configure layer behavior.

Format: `layerrule=layer_name:value,parameter:value`

| Parameter | Values | Description |
|-----------|--------|-------------|
| `layer_name` | string | Layer name (regex) |
| `animation_type_open` | slide/zoom/fade/none | Open animation |
| `animation_type_close` | slide/zoom/fade/none | Close animation |
| `noblur` | 0/1 | Disable blur |
| `noanim` | 0/1 | Disable animation |
| `noshadow` | 0/1 | Disable shadow |

Examples:

```ini
layerrule=noblur:1,layer_name:selection
layerrule=noanim:1,layer_name:popup
```

Get layer name: `mmsg -e`

## Monitor Rules

Configure monitors.

Format: `monitorrule=name,mfact,nmaster,layout,transform,scale,x,y,width,height,rate`

Transform: 0=normal, 1=90°, 2=180°, 3=270°, 4=vertical flip, etc.

Example:

```ini
monitorrule=eDP-1,0.55,1,tile,0,1,0,0,1920,1080,60
monitorrule=HDMI-A-1,0.55,1,tile,0,1,1925,0,2560,1440,144
```

## Tearing(Game mode)

| Setting | Type | Default | Description |
|---------|------|---------|-------------|
| `allow_tearing` | integer | 0 | Allow tearing,refer to [tearing](#tearinggame-mode) |
| `force_tearing` | integer | 0 | Set window to tearing state,refer to [tearing](#tearinggame-mode) |

```ini
allow_tearing=1
windowrule=force_tearing:1,title:vkcube
```

## Virtual Monitors

Create virtual displays.

IPC commands:

```bash
mmsg -d create_virtual_output
mmsg -d destroy_all_virtual_output
```

Configure with wlr-randr:

```bash
wlr-randr --output HEADLESS-1 --pos 1921,0 --custom-mode 1920x1080@60Hz
```

## Scratchpad

### Sway like scratchpad

```ini
# put a window in scratchpad link
bind=SUPER,i,minimized,

# Go through all the bookmarks
bind=ALT,z,toggle_scratchpad

# move out a window from scratchpad link
# you can also you togglefloating move out
bind=SUPER+SHIFT,I,restore_minimized
```

### Named Scratchpad

You can bind a named scratchpad by fixing the title or appid of a window. In this way, whether the window is launched in advance or not, as long as you press this shortcut key, it will toggle show or hide.

The application you use needs to have the ability to set an appid or title, such as `kitty-T` or `st -c` to fix the title or appid.

```ini
# toggle_named_scratchpad,[appid],[title],[cmd]
bind=alt,h,toggle_named_scratchpad,st-yazi,none,st -c st-yazi -e yazi
windowrule=isnamedscratchpad:1,width:1280,height:800,appid:st-yazi
```

## IPC (mmsg)

Control MangoWC externally.

Basic usage:

```bash
mmsg -d killclient
mmsg -d spawn,foot
mmsg -g  # Get info
mmsg -w  # Watch changes
```

Common commands:

- `reload_config`: Reload config
- `quit`: Exit MangoWC
- `killclient`: Close window
- `focusdir,left`: Focus direction
- `view,1`: Switch to tag 1
- `setlayout,scroller`: Change layout

See [IPC documentation](/docs/features/ipc) for details.

## Screen Scale

### Not use global scale(recommended)

If you do not use xwayland apps, you can just use monitor rule or wlr-randr to set a global monitor scale.

If you are using some xwayland apps, I don't suggest you to set monitor scale.

You can set scale like this , such as 1.4 factor.

Dependencies:

```bash
yay -S xorg-xrdb
```

In config file.

```ini
env=QT_AUTO_SCREEN_SCALE_FACTOR,1
env=QT_WAYLAND_FORCE_DPI,140
```

In autostart.

```bash
echo "Xft.dpi: 140" | xrdb -merge
gsettings set org.gnome.desktop.interface text-scaling-factor 1.4
```

### 3.edit autostart.sh

```bash
# start xwayland
/usr/sbin/xwayland-satellite :11 &
# scale 1.4 for xwayland
sleep 0.5s && echo "Xft.dpi: 140" | xrdb -merge
```
